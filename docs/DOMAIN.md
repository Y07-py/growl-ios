# Growlのドメイン
ここでは、ビジネス的な視点から見たGrowlが解決しようとする問題領域について、コアドメイン、サブドメインとに分けて定義します。

## コアドメイン
Growlの価値の核となるドメインです。

### 嗜好解析・ドメイン
- **責務**: 飲食関連の写真を解析し、ユーザーの「好き」を構造化されたデータに変換する。
- **主要概念**: 嗜好プロファイル、画像特徴量、食の属性（雰囲気、ジャンル、価格帯）

### 推薦エンジン・ドメイン
- **責務**: 解析されたプロファイルと、外部の飲食店データ・レシピデータを照らし合わせ、最適な「食体験」をマッチングする。
- **主要概念**: 推薦スコアリング、フォールバック戦略（高評価店）、グループ推薦ロジック

## サブ・ドメイン
サブドメインではGrowlのコアドメインの次に重要な問題領域を定義します。

### コンテンツ・ドメイン
- **責務**: 外部APIから取得した店舗情報や、自前のレシピデータの管理
- **主要概念**: 飲食店情報、メニュー、レシピ、営業ステータス

### ソーシャル・ドメイン
- **責務**: ユーザーによる食体験の投稿・共有・他者どの繋がり
- **主要概念**: 投稿、共有設定、ユーザー間の嗜好プロファイルの共有

### アイデンティティ・ドメイン
- **責務**: ユーザー認証と認可、およびサブスクの管理
- **主要概念**: ユーザーアカウント、権限管理、決済管理
